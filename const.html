Can You Add New Keys to a const Object?

‚úÖ YES, You Can!
// Creating a const object
const person = {
  name: "John",
  age: 25
};

// ‚úÖ Adding a NEW key (property) to const object
person.city = "New York";

// ‚úÖ Adding another new key with bracket notation
person['job'] = "Developer";

// ‚úÖ Adding a key with a dynamic name
const newKey = 'email';
person[newKey] = "john@example.com";

// ‚úÖ Result: Object now has ALL new properties!
console.log(person);
// { name: "John", age: 25, city: "New York", job: "Developer", email: "john@example.com" }

What You CAN Do with const Objects

  • ‚úÖ Add new properties/keys
  • ‚úÖ Modify existing property values
  • ‚úÖ Delete properties (with delete operator)
  • ‚úÖ Change property descriptors
  • ‚úÖ Use Object methods (freeze, seal, etc.)

What You CANNOT Do with const Objects

  • ‚ùå Reassign the variable itself
  • ‚ùå Point it to a different object
  • ‚ùå Redeclare it in the same scope

  • ‚úÖ You can remove items from the suitcase
  • ‚úÖ You can rearrange items in the suitcase
  • ‚ùå But you cannot swap the suitcase for a different one

Why This Works: Understanding const

const means "constant reference", not "constant value".

When you do:

const obj = { a: 1 };

What's happening:

  • A new object is created in memory at address 0x1234
  • The variable obj stores the reference (memory address 0x1234)
  • const means this reference cannot change
  • But the object at 0x1234 can be modified freely!

So when you add a new key:

obj.b = 2;

You're modifying the object at address 0x1234, not changing the reference stored in obj.
